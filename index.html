  1. TCP
    • Transmission Control Protocol
    • Verbindingsgericht protocol wat zorgt voor een betrouwbare afhandeling van gegevens tussen client en server.
  2. IP-Adres
    • Internet Protocol Adres
    • Een uniek adres waarmee een device te identificeren is binnen een netwerk.
  3. DNS
    • Domain Name Server
    • DNS vertaald IP's in text die voor mensen makkelijk te onthouden is, bijvoorbeeld google.com
  4. Router
    • Een router verbind verschillende netwerken met elkaar en stuurt datapakketen naar de juiste client.
  5. Webserver
    • Een programma wat verzoeken verwerkt en juiste data teruggeeft over een netwerk.
  6. HTML
    • Hyper Text Markup Langugage
    • Een opmaaktaal die word gebruikt voor webdocumenten.
  7. HTML-Tag
    • Een tag markeert een gedeelte van een pagina om op te maken/formatten.
  8. Hoe weet een browser wat voor soort bestand door de server wordt geretourneerd?

    In de HTTP headers staat de extensie en deze staat ook in de bestandsnaam.

  9. Wat is het doel van keywords bij een website? Wie of wat gebruikt deze?

    Zoekmachines gebruiken keywords om sites te laten zien bij bepaalde zoekopdrachten.

  10. Wat is het doel van een description? Waar zal deze, wanneer je je site publiceert, te zien zijn?

    De description word weergeven onder links in zoekmachines zodat mensen snel kunnen zien wat de pagina bevat.

  11. Klik in je browser met je rechter muisknop op je pagina, en kies ‘Pagina weergeven’ (of Ctrl+U). Wat zie je?

    De source van de pagina.

  12. Klik op dezelfde wijze, en selecteer ‘Element inspecteren’ (of Ctrl+Shift+I). Beweeg met je muis over het scherm. Wat zie je? Wat is het nut hiervan?

    Je kunt van alle aparte html elementen de eigenschappen bewerken in real time.
